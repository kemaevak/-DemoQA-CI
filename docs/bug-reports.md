они всё равно оформлены как баг-репорты: с точки зрения пользователя
поведение остаётся некорректным, а для портфолио важен сам навык
оформления.

## BUG-001

**Заголовок:** Radio Button: вариант «No» недоступен для выбора

**Серьёзность:** Major | **Приоритет:** High

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/radio-button
2. Попытаться выбрать вариант «No»

**Ожидаемый результат:** вариант выбирается, на странице появляется
текст «You have selected No».

**Фактический результат:** радиокнопка отключена (атрибут `disabled`),
выбрать вариант невозможно. Из трёх предложенных ответов доступны
только два.

**Вложение:** [bug-001-radio-no-disabled.png](screenshots/bug-001-radio-no-disabled.png)

## BUG-002

**Заголовок:** Text Box: форма отправляется пустой, обязательных полей нет

**Серьёзность:** Minor | **Приоритет:** Medium

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/text-box
2. Не заполняя ни одного поля, нажать Submit

**Ожидаемый результат:** обязательные поля подсвечиваются, выводится
сообщение о необходимости заполнить форму.

**Фактический результат:** никакой реакции — ни сообщений валидации,
ни обозначения обязательных полей. Пользователь не понимает, произошло
что-то после нажатия кнопки или нет.

## BUG-003

**Заголовок:** Links: опечатка «staus» в сообщении о статусе ответа

**Серьёзность:** Trivial | **Приоритет:** Low

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/links
2. Нажать ссылку Created (или любую другую API-ссылку)

**Ожидаемый результат:** сообщение «Link has responded with status 201
and status text Created».

**Фактический результат:** сообщение «Link has responded with **staus**
201 and status text Created» — пропущена буква «t» в первом слове
«status».

**Вложение:** [bug-003-links-typo.png](screenshots/bug-003-links-typo.png)

## BUG-004

**Заголовок:** Alerts: опечатка в id кнопки prompt-диалога — «promtButton»

**Серьёзность:** Trivial | **Приоритет:** Low

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/alerts
2. Открыть инспектор и найти кнопку prompt-диалога

**Ожидаемый результат:** id кнопки — `promptButton`, единообразно
с элементом результата `promptResult`.

**Фактический результат:** id кнопки — `promtButton` (пропущена «p»).
На работу пользователя не влияет, но затрудняет автоматизацию
и сопровождение: рядом лежат `promtButton` и `promptResult`
с разным написанием.

## BUG-006

**Заголовок:** Practice Form: в поле Select Picture загружается файл любого типа

**Серьёзность:** Minor | **Приоритет:** Medium

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/automation-practice-form
2. В поле Select Picture выбрать текстовый файл (.txt)

**Ожидаемый результат:** файл отклоняется — поле предназначено для
изображений; в диалоге выбора файла предлагаются только графические
форматы.

**Фактический результат:** текстовый файл принимается без
предупреждений. У поля отсутствует атрибут `accept`, ограничений
по типу и размеру файла нет.

## BUG-007

**Заголовок:** Web Tables: принимается запись с возрастом 0 и зарплатой 0

**Серьёзность:** Minor | **Приоритет:** Medium

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/webtables
2. Нажать Add
3. Заполнить форму корректными данными, указав Age = 0 и Salary = 0
4. Нажать Submit

**Ожидаемый результат:** значения отклоняются как недопустимые —
сотрудника с возрастом 0 лет и зарплатой 0 существовать не может.

**Фактический результат:** запись сохраняется и появляется в таблице.
Валидация проверяет только формат (цифры), но не осмысленность
значений.

**Вложение:** [bug-007-webtables-zero-values.png](screenshots/bug-007-webtables-zero-values.png)

## BUG-008

**Заголовок:** Рекламные блоки смещают контент и мешают взаимодействию с элементами

**Серьёзность:** Minor | **Приоритет:** Medium

**Шаги воспроизведения:**
1. Открыть любую страницу сайта, например https://demoqa.com/buttons
2. Дождаться загрузки рекламы

**Ожидаемый результат:** контент страницы стабилен, все элементы
доступны для взаимодействия сразу после загрузки.

**Фактический результат:** рекламные блоки (до 970x250) вставлены
между элементами страницы и подгружаются с задержкой, из-за чего
контент «прыгает» уже после загрузки. Кликнуть по элементу в этот
момент невозможно или клик попадает в рекламу. В автотестах проекта
рекламные домены пришлось блокировать (см. `tests/conftest.py`).

## BUG-009

**Заголовок:** Text Box: при некорректном email нет текста ошибки

**Серьёзность:** Minor | **Приоритет:** Low

**Шаги воспроизведения:**
1. Открыть https://demoqa.com/text-box
2. В поле Email ввести строку, не являющуюся адресом (например «abc»)
3. Нажать Submit

**Ожидаемый результат:** рядом с полем выводится понятное сообщение,
например «Введите корректный email».

**Фактический результат:** поле подсвечивается красной рамкой, но
никакого текста с объяснением нет. Пользователь должен сам догадаться,
что именно не так с введённым значением.

**Вложение:** [bug-009-email-no-error-text.png](screenshots/bug-009-email-no-error-text.png)
